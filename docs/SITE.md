# אתר המותג (Public Site) — סטטוס

תשתית האתר הציבורי סביב הסטודיו. **הסטודיו עצמו** (מערכת עיצוב התכשיטים —
`/studio` וכל ה־`/api`) מפותח בנפרד ולא נגעתי בו כאן.

## מה נבנה

עמודים ציבוריים בעברית (RTL), mobile‑first, Tailwind בלבד — משתמשים בתשתית
Next.js/Cloudflare/Supabase הקיימת. כולם קבוצת ראוטים `(site)` עם Header/Footer
משלה; הסטודיו נשאר על ה־root layout בלבד.

| נתיב | תיאור |
|---|---|
| `/` | דף בית — Hero, רצועת ערך, חומר ומוצר, תקציר "איך זה עובד", CTA לסטודיו |
| `/how-it-works` | ארבעת שלבי התהליך (תיאור → יצירה → בדיקת ייצור → ייצוא) |
| `/gallery` | שש דוגמאות סגנון (איורים דקורטיביים, מסומנים במפורש כהמחשה) |
| `/faq` | שאלות נפוצות עם אקורדיון; תשובות מעוגנות בעובדות המוצר האמיתיות |
| `/contact` | טופס יצירת קשר שמרכיב `mailto:` (בלי backend/מיילים) + כתובת ישירה |

`/` הופנה בעבר ל־`/studio` (redirect) — הוחלף בדף בית אמיתי; הקישור "התחלת עיצוב"
מוביל לסטודיו.

## קבצים

- `src/app/(site)/…` — layout + חמשת העמודים
- `src/components/site/` — `SiteHeader`, `SiteFooter`, `PatternMark`, `FaqAccordion`, `ContactForm`
- `src/lib/site.config.ts` — פרטי קשר + ניווט (מקור אמת יחיד)
- `src/i18n/he.ts` — כל המחרוזות תחת `he.site` (לפי מוסכמת קובץ ה־i18n היחיד)

## לגל — נקודות לעדכון

- `src/lib/site.config.ts`: `contactEmail` הוא placeholder (`hello@forme.co.il`) —
  להחליף בכתובת האמיתית; `instagram` = `null` עד שיש חשבון.
- הגלריה היא איורים דקורטיביים בלבד (מסומן בעמוד) — להחליף בעיצובים אמיתיים כשירצה.
- מחיר/זמן אספקה לא הומצאו — ה־FAQ מפנה ל"צרו קשר לפרטים".
